The order your styles are declared in and linked to matters!

Follow the order that is declared, the latest wins, but specificity is more important. When there's conflict in specificity, the latest wins.

Specificity

Which rules applies when there's a conflict, when not conflicting every property applies. The more specific, more applies.

ID > Class > Element

ID (100x) > Class, Attribute & Pseudo Class (10x) > Element & Pseudo Element (1x)

Specificity calculator: https://specificity.keegan.st/

Inline style

Is even MORE important than ID, but not much used, avoid inline styling.

!important - overrides everything

https://developer.mozilla.org/en-US/docs/Web/CSS/Specificity#the_!important_exception

button {

color: red !important;

}
